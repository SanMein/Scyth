# Scyth // Σκύθ

## Общее назначение

Бот предназначен для автоматизации **модерации**, **логирования действий**, **выдачи ролей**, **контроля приглашений** и **системы предупреждений** на Discord-сервере.  
Ориентирован на использование административным составом, обладающим специальными ролями.

---

## 1. Система прав доступа

- Команды доступны только пользователям с **одной из трёх указанных ролей**:
  - `ἄλφα` (ID: 1447513054844555336)
  - `Генеральный Директор` (ID: 1349365796970954834)
  - `Операционный директор` (ID: 1349365796970954833)
- При отсутствии необходимой роли бот отклоняет выполнение команды с соответствующим уведомлением.

---

## 2. Команды модерации

### 2.1. `//ban` – блокировка пользователя
**Синтаксис:** `//ban <ID пользователя> [длительность] [причина]`  
**Описание:**  
- Возможна **перманентная** или **временная** блокировка.  
- При временной блокировке бот автоматически разблокирует пользователя по истечении срока.  
- Поддерживаются форматы длительности: `m` (минуты), `h` (часы), `d` (дни), `w` (недели), `t` (минуты, аналог m), `y` (годы).  
- Уведомление о блокировке отправляется пользователю в ЛС (если доступно).  
- Все действия логируются.

### 2.2. `//unban` – снятие блокировки
**Синтаксис:** `//unban <ID пользователя> [причина]`  
**Описание:**  
- Разблокирует пользователя, если он находится в списке забаненных.  
- Уведомление о разблокировке отправляется пользователю в ЛС.

### 2.3. `//kick` – выгон пользователя
**Синтаксис:** `//kick <@пользователь> [причина]`  
**Описание:**  
- Удаляет пользователя с сервера без блокировки.  
- В сообщении кика пользователю направляется ссылка для повторного входа и контакт администрации.

### 2.4. `//tout` – таймаут
**Синтаксис:** `//tout <@пользователь> <длительность> [причина]`  
**Описание:**  
- Ограничивает возможность пользователя писать в чат на указанное время.  
- Максимальная длительность – 28 дней.

### 2.5. `//clear` – массовое удаление сообщений
**Синтаксис:** `//clear <количество>`  
**Ограничения:**  
- Максимум 1000 сообщений за раз.  
- Удаляются сообщения только в канале вызова команды.  
- Бот самостоятельно удаляет своё подтверждающее сообщение через 3 секунды.

## 3. Система предупреждений (Warns)

### 3.1. `//warn` – выдача предупреждения
**Синтаксис:** `//warn <@пользователь> <длительность> [причина]`  
**Описание:**  
- Предупреждения сохраняются в файле `warnings.json`.  
- Каждое предупреждение имеет причину, длительность, дату и модератора.  
- **Автоматические санкции при накоплении:**
  - 3 предупреждения → таймаут на 30 минут  
  - 6 предупреждений → таймаут на 3 часа  
  - 10 предупреждений → перманентный бан

### 3.2. `//unwarn` – снятие предупреждения
**Синтаксис:** `//unwarn <@пользователь> [номер предупреждения]`  
**Описание:**  
- Если номер не указан – удаляется **последнее** предупреждение.  
- Удаление логируется.

### 3.3. `//clwarn` – полная очистка предупреждений
**Синтаксис:** `//clwarn <@пользователь>`  
**Описание:**  
- Удаляет **все** предупреждения у указанного пользователя.

## 4. Логирование и аудит

### 4.1. `//logs` – просмотр логов пользователя
**Синтаксис:** `//logs <@пользователь>`  
**Описание:**  
- Показывает последние **15 действий** модерации в отношении пользователя.  
- Включает: тип действия, модератора, причину, дату.

### 4.2. Автоматическое логирование всех действий
Каждое из действий (`ban`, `unban`, `kick`, `tout`, `warn`, `unwarn`, `clwarn`, `clear`) фиксируется в:
- файле `moderation_logs.json` (структурированное хранение по пользователям)
- специальном **канале логов** (ID: 1399890569165275348) в виде embed-сообщения с подписью бота.

---

## 5. Система приглашений (Invite Tracker)

### 5.1. Отслеживание приглашений
- При запуске бот сохраняет список всех действующих инвайтов.  
- При заходе нового участника бот:
  - сравнивает старый и новый список приглашений
  - определяет, по какому инвайту зашёл пользователь
  - фиксирует **пригласившего**
  - подсчитывает общее количество приглашений пригласившего

### 5.2. Уведомление о входе
В заданный канал (ID: 1349365797515956225) отправляется embed-сообщение:
- кто пригласил
- кто зашёл
- возраст аккаунта новичка (годы/месяцы/дни)
- количество инвайтов у пригласившего

### 5.3. Обновление инвайтов
При создании нового инвайта бот автоматически сохраняет его в `invites.json`.

## 7. Вспомогательные функции

- **Парсинг длительности** (`parse_duration`) – преобразует строки вида `5m`, `2h`, `1d` в объект `timedelta`.
- **Асинхронное хранение данных** – все JSON-файлы читаются и записываются через `aiofiles`.
- **Единое оформление embed-сообщений** – все уведомления содержат подпись с текстом `Scyth // Σκύθ` и иконкой.
- **Обработка ошибок** – команды перехватывают исключения `Forbidden`, `HTTPException`, `ValueError` и возвращают понятные сообщения.

## Заключение

Бот реализует **полный цикл модерации** Discord-сервера: от выдачи предупреждений и временных блокировок до автоматической выдачи ролей, отслеживания приглашений и централизованного логирования. Система спроектирована для использования административной командой с чёткими уровнями доступа и минимальным участием человека в рутинных операциях.
